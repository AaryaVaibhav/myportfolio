AWS Capstone Project

Deploy a Highly Available WordPress Application using AWS Services

Introduction:

This project was done while trying to gain some handson exposure to use the newly learnt AWS Knowledge. In this Mini-Project - I will create a highly available (HA), scalable and fault-tolerant deployment of the WordPress application. I will deploy the WordPress application in such a way that the application server, load balancer and database will scale independently of one another. I will also deploy the application's components like the web server and database into two availability zones to distribute it and guard against failure of the anyone availability zone. The WordPress application will be deployed in a stateless fashion so that I can add or remove web application servers in response to the requests flowing into the system. Finally, I will create a CloudFront distribution as CDN and change the configuration of WordPress. The above diagram depicts the goal/architecture of the Mini-Project.

Project Details:

There are multiple steps to be followed while building this application setup and they are mentioned below:

  • Step 1 - Setting up the architecture

    I will first setup the application environment by creating a VPC followed by creating subnets within the VPC. A public subnet to host my web application. For that, I will launch an EC2 Instance. Also, a Private subnet to deploy my Amazon RDS MySQL database. Then, I will set up the connection between RDS and EC2, then install the WordPress website on the instance. Finally, you will start hosting a simple WordPress website on an EC2 instance.

    More detailed information on above setup can be found on my blog Here.

  • Step 2 - Create Application Load Balancer

    Further I will create a snapshot of the earlier created instance and go ahead by doing the settings for attaching them to the ALB.

  • Step 3 - Create Auto Scaling Group

    The next step is creating a Auto Scaling group and select the earlier created configuration for the instance. Make the necessary security group settings and then set the desired/Minimum/Maximum capacity and attach it to the created ALB.

  • Step 4 - Create CloudFront Distribution

    Visit CloudFront console, and click Create distributions, make the settings as per the architecture.Once all the setup is complete, we should be able to acess our wordpress application through CloudFront.
